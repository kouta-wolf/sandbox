# Rails Model層のvalidatesの挙動を純Rubyで考えた開発

### 目的
- ユーザー登録時における「ユーザー名」と「パスワード」のvalidationを自前で実装する(match)
- Railsのvalidates機能に頼らず、その裏側の正規表現等の処理を理解する

### 仕様
#### ユーザー名
- 3文字以上、15文字以内
- 半角英数字のみ

#### パスワード
- 8文字以上
- 英大文字、英小文字、数字をそれぞれ1文字以上含む

### 実行方法
```
$ gem install rspec
$ rspec spec/main_spec.rb --format documentation
```

### 学んだ点
- 正規表現の活用: /[^a-zA-Z0-9]/ を使って全角文字を判定できるように設定
- Ruby的記法: 後置ifを使って可読性とメンテナンス性を両立
- countメソッドの使用：matchのみならず含まれているなら数を数えるのでも可能ということ

### テストケース
UserValidator
  .validate
    全ての条件を満たすときに有効であること
    ユーザー名のバリデーション
      ユーザー名が短すぎると無効であること
      ユーザー名が長すぎると無効であること
      ユーザー名に全角文字があると無効であること
    パスワードのバリデーション
      パスワードが短すぎると無効であること
      パスワードに英大文字が含まれていないと無効であること
      パスワードに英小文字が含まれていないと無効であること
      パスワードに数字が含まれていないと無効であること

### 工夫した点
- テストケースはループで回すことも出来ますが冗長性があることで分かりやすい設計にしています（ただfactory_bot_railsを使って最初のresultは直すことはできます）
- 実装自体は最初は冗長でもテストを回すことで効率の良い書き方を後から模索できるためテストを複数回実行しながらリファクタリングを実施した
