# ショッピングカードの価格計算

### 仕様
- 軽減税率は飲料食品に適用すること(今回はcategoryは適用するか否かで処理)
- 5000円以上のときは500円割引されるセール中

### 目的
- RailsアプリのModel層で書く「計算処理」をRubyで実装と検証
- TDDを意識
- 軽減税率の適用も考慮して処理

### テーブル設計
name: "本", price: 1000, category: :normal
name: "りんご", price: 100, category: :reduced

categoryは本来はbooksとかfoodsとなり別テーブルになるが今回は軽減税率のみに焦点を絞っているためcategoryには軽減税率を適用するかどうかを埋め込む
本来ならさらにenumで管理しintegerで管理すると良いかも

### 実装手順
- まずはテストを書く。テストはロジックに関わるためデータ自体を書かない
- テストのFailuresを参照しながら実装を行っていく
- 途中途中でbinding.irbで止めながら数値を確認する
- returnはハッシュで渡す

### 工夫ポイント
- 将来の税率変更のために定数で盛り込んだこと（ただしテストは書き直し）
- DB設計も考慮して商品に税率をもたせなかったこと
- 可読性向上のために変数名は分かりやすいものにした
- テストケースで境界値を盛り込んだ

### 学んだこと
- RSpecやpryは純Rubyでも使うことが可能なこと
- returnはハッシュでも可能なこと
- テスト駆動はやっぱり便利なこと
